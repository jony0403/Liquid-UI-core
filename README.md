# Liquid-UI-core

[Project Proposal] 유동적 사용자 경험을 위한 반응형 액체 UI (Liquid User Interface)
1. 기획 의도 및 배경 (Why?)
"정적인 웹(Static Web)의 종말과 정보 과부하의 해결"

현황 (As-Is): 현재의 웹 인터페이스는 '사각형(Grid)'에 갇혀 있다. 모든 사용자에게 똑같은 크기, 똑같은 텍스트를 보여준다. 이는 지루할 뿐만 아니라, 시각적 피로도를 높이고 정보 습득의 효율을 떨어뜨린다.

문제점 (Pain Point):

Visual Boredom: 천편일률적인 레이아웃은 사용자에게 어떤 자극도 주지 못한다.

Information Overload: 너무 많은 정보가 한 번에 쏟아진다. 확대(Zoom)조차 귀찮은 과정이다.

제안 (To-Be): **'액체(Liquid)'**의 물성을 UI에 도입한다. 그릇에 따라 형태가 변하는 물처럼, 사용자가 필요로 하는 핵심 정보만 유동적으로 변형(Morphing)되어 강조되는 새로운 차원의 UX를 제안한다.

2. 프로젝트 목표 (Goal)
"심미성(Aesthetics)과 기능성(Functionality)의 완벽한 결합"

Visual Interaction: 정적인 웹사이트를 동적인 유기체로 전환하여 시각적 몰입감을 극대화한다. (Gooey Effect, Distortion 기술 적용)

Adaptive Summarization: 정보의 홍수 속에서 AI 요약 기술과 UI 확대를 연동하여, 사용자가 원하는 핵심 정보만 '떠오르게' 만든다.

Experimental Usability: 차세대 웹 인터랙션의 실현 가능성과 기술적 연계성을 검증한다.

3. 핵심 타겟 (Target Audience)
"싫으면 쓰지 마"가 아니다. **'니즈(Needs)가 확실한 집단'**을 타겟팅한다.

Primary Target (핵심 사용자):

Dopamine Seekers (도파민 추구형 Z세대/알파세대): 정적인 텍스트 읽기를 거부하고, 즉각적이고 화려한 인터랙션에 반응하는 젊은 층. 기존 웹서핑을 '지루하다'고 느끼는 얼리어답터.

Secondary Target (기능적 사용자):

Time-Poor Professionals (시간이 부족한 현대인): 뉴스나 아티클을 정독할 시간이 없어, 핵심만 직관적으로(크고 명확하게) 보고 싶은 직장인 및 학생.

Low Vision Users (잠재적 배려 대상): 복잡한 확대/축소 조작 없이, 시선이 머무는 곳의 정보가 자동으로 커지는 기능을 필요로 하는 저시력자.

4. 주요 기능 및 시나리오 (Key Features)
Liquid Morphing Engine (핵심 기술):

스크롤이나 마우스 호버 시, div 박스와 이미지가 액체처럼 일렁이며 재배치된다.

딱딱한 그리드가 아닌, 유기적인 움직임으로 시각적 즐거움을 제공한다.

Focus & Flow (정보 전달):

사용자가 특정 키워드나 영역에 관심을 보이면(호버/클릭), 주변의 불필요한 정보는 액체처럼 흘러내리고(축소), 핵심 요약 정보가 물방울이 합쳐지듯 크게 확대된다.

"뉴스를 읽는 것"이 아니라, "핵심 정보를 흡수하는" 경험을 제공한다.

5. 기대 효과 (Expected Effect)
Paradigm Shift: "웹사이트는 멈춰 있어야 한다"는 고정관념을 탈피, 동적 웹 디자인의 새로운 가능성(Potential) 제시.

Enhanced UX: '재미(Fun)'가 곧 '사용성(Usability)'으로 이어질 수 있음을 증명. 지루한 정보 습득 과정을 엔터테인먼트의 영역으로 확장.
